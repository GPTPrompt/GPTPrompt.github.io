## 获得更好结果的六项策略


### 撰写清晰的指导说明
这些模型无法读取您的思想。如果输出太长，请要求简要回复。如果输出过于简单，请要求专业水平的写作。如果您不喜欢格式，请展示您希望看到的格式。模型越少猜测您想要什么，您得到的可能性就越大。


### 策略1：在查询中包含详细信息以获得更相关的答案

为了获得高度相关的响应，请确保请求提供任何重要的详细信息或上下文。 否则，你将让模型来猜测你的意思。

| 更差        | 更好 |  
| :------------ | :--------- | 
| 如何在 Excel 中添加数字？        |    如何在 Excel 中添加一行美元金额？ 我想对整张行自动执行此操作，所有总计都在右侧名为“总计”的列中结束。    | 
| 谁是总统？     |    谁是 2021 年墨西哥总统？选举频率如何？     |  
| 编写代码来计算斐波那契数列。 |    编写一个 TypeScript 函数来高效计算斐波那契数列。 自由地注释代码以解释每部分的作用以及为什么这样编写。         |   
| 总结会议记录。        | 用一个段落总结会议记录。 然后写下演讲者的 Markdown 列表以及他们的每个要点。 最后，列出发言人建议的后续步骤或行动项目（如果有）。 | 
	

### 策略2：要求模型采用一种角色

系统消息可用于指定模型在其回复中使用的角色。

|    角色      | 内容 |  
| :------------ | :--------- | 
|   系统  | 当我请求帮助撰写某事时，您将回复一篇文档，其中每一段至少包含一个笑话或轻松的评论。 |  
|   用户  | 给我的螺栓供应商写一封感谢信，感谢他们按时并在短时间内交付。这使我们能够完成一份重要的订单。| 


### 策略3：使用分隔符清晰地指示输入的不同部分

分隔符，如三重引号、XML 标记、章节标题等，可以帮助标明要以不同方式处理的文本部分。

|    角色      | 内容 |  
| :------------ | :--------- | 
| 用户  | 用三重引号分隔的文本写一个俳句摘要。<br/> """在此插入文字""" |  

|    角色      | 内容 |  
| :------------ | :--------- | 
| 系统  | 您将获得一对关于同一主题的文章（用 XML 标记分隔）。首先总结每篇文章的论点。然后指出哪篇文章的论点更有说服力，并解释原因。 |  
| 用户  | `<article>`在此处插入第一篇文章`</article>` <br/><br/> `<article>`在此处插入第二篇文章`</article>`| 


|    角色      | 内容 |  
| :------------ | :--------- | 
| 系统  | 您将获得论文摘要和建议的标题。 论文标题应该让读者清楚地了解论文的主题，但也应该引人注目。 如果标题不符合这些标准，请提出 5 个替代方案。 |  
| 用户  | 摘要：在此插入摘要 <br/><br/>  标题：在此插入标题 |

### 策略5：提供例子

通常，提供适用于所有示例的一般性说明比通过示例演示任务的所有排列组合更为高效，但在某些情况下，提供示例可能更容易。例如，如果您打算让模型复制一种难以明确描述的用户查询响应风格。这被称为“少样本学习”提示。

|    角色      | 内容 |  
| :------------ | :--------- | 
| 系统  | 以一致的风格回答。 |  
| 用户  | 教我耐心。 | 
| 助手  | 刻出最深谷的河流源自一处不起眼的泉水；最宏伟的交响乐始于一个音符；最复杂的挂毯以一根孤独的线开始。 | 
| 用户  | 教我关于海洋。 | 


### 策略6：指定所需的输出长度

您可以要求模型生成具有给定目标长度的输出。目标输出长度可以按照单词、句子、段落、项目符号等的计数来指定。但请注意，指示模型生成特定数量的单词的精度可能不高。模型可以更可靠地生成具有特定数量段落或项目符号的输出。

|    角色      | 内容 |  
| :------------ | :--------- | 
| 用户  | 用大约50个字总结三重引号分隔的文本。 <br/><br/> """在此插入文字""" |  

|    角色      | 内容 |  
| :------------ | :--------- | 
| 用户  | 用2个段落总结三重引号分隔的文本。 <br/><br/> """在此插入文字""" |  

|    角色      | 内容 |  
| :------------ | :--------- | 
| 用户  | 将由三引号分隔的文本总结为 3 个要点。 <br/><br/> """在此插入文字""" |
